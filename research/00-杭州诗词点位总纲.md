# 《诗径》杭州诗词点位 · 考据总纲

> 本文件是杭州城诗词点位搜集与考证的**作战地图**与**质量标准**。
> 阶段定位:只做内容考据,不写代码。逐点查证、逐句溯源,成熟后再落成 `data/*.json` 诗径。
> 最后更新:2026-06

---

## 一、为什么是杭州,为什么这么做

杭州(尤其西湖)是中国诗词地理密度最高的城市之一:自唐李泌、白居易,经五代吴越,到南宋建都临安,再到明清乾隆题景,**一千余年文人题咏层层叠压在同一片湖山上**。这正是《诗径》"走到诗里、时空叠合"的最佳土壤。

但密度高也意味着**陷阱多**:大量诗词只是"咏西湖"的泛咏,并非确指某一具体点位。本工程的全部价值,在于把"确指此地此景此事"的作品,与"泛咏西湖"的作品严格区分——**宁可留白,不滥触发**。

## 二、考据标准(沿用并强化现有 A/B 分级)

### 可信度分级
| 级 | 含义 | 准入 |
|---|---|---|
| **A 确指** | 诗文/碑记明确为此点位、此景、此事而作,有传世文献可考 | 默认收录 |
| **B 相关** | 确与此地相关(咏及此地/长眠于此),但非专为此点此景而作 | 谨慎收录,需在卡片注明 |
| ✗ 泛咏 | 仅"咏西湖/咏杭州",无法确指具体点位 | **不收录** |

### 每条诗文必备字段(对标 poems.json 的 p2 过溪亭标准)
- `text` 诗文原文(注明通行本/善本,标"待纸本终校")
- `author` / `dynasty` / `source` 作者/朝代/篇名
- `year` 创作年份(不确定标 `year_approx`)
- `credit` A/B 分级
- `annotation` 考据说明:**为何确指此地**(一手文献依据)
- `vernacular` 白话translation
- `ref` 文献出处(具体到卷次/善本/碑帖)
- (可选)`relic` / `relic_image` 关联文物/真迹

### 坐标溯源标准(WGS-84 全程不偏移)
- `coord_system` = WGS-84
- `coord_source` = `osm`(与 OSM 节点交叉核验) / `field`(实地实测) / `estimate`(史料估算待测)
- `coord_accuracy_m` / `radius_verified`
- 拿不到确指坐标的点 → 标 estimate,**绝不编造精确值**

### 核校工具
- 坐标:`geocheck.py`(overpass / gcj2wgs / dist 等子命令)+ survey.html 实地打点
- 文献:优先中华书局点校本、《淮海集》《苏轼诗集》《苏轼文集》《全唐诗》《全宋词》;在线优先 wikisource / guwendao 逐字核校,标注来源

### 共享点位规则(一个地标可挂多条诗径)

一个地标因身份多元,可同时归入多条主题诗径(如吴山·有美堂"左眺钱江、右瞰西湖",同挂④钱塘江畔诗径与⑤西湖名胜诗签)。这是**有意为之的产品机制**,不是重复数据。规则如下:

1. **id 复用、跨径一致**:同一物理地标在不同 `data/*.json` 中沿用同一 `id`(如 `c4`)。便于全局识别"这是同一个点"。
2. **坐标单一来源(single source of truth)**:同一 id 的 `lat/lng/coord_system/coord_source/coord_accuracy_m` 在所有 json 中**必须完全一致**。改坐标时**所有挂载处一并改**,严防同点在不同 json 里坐标漂移。本总纲及各区 research md 的"坐标汇总表"为该点坐标的权威记录,以它为准。
3. **诗作/文案可因径而异**:同一地标在不同诗径里可呈现不同的 `scene_hint`/选诗/`subtitle`,以贴合各径的叙事主题(望江 vs 望湖)——这正是多挂的价值。但坐标与考据结论(credit 级别、公案)须一致。
4. **触发不冲突**:用户当下只激活一条诗径,GPS 围栏只对当前诗径的点判定,走到共享地标只弹当前径的卡,不会重复触发。
5. **维护备注**:在该点主属区的 research md 条目里标注"★双挂/多挂设计(有意为之,勿删)"+ 挂载清单,防止后续维护误删。
6. **未来候选**:断桥、苏堤、雷峰塔等湖区地标日后亦可能身兼多线,统一适用本规则。

## 五、工作流程(每个点位)

1. **定篇**:确认有无确指此点的名篇(查篇名/序/本事)
2. **溯源**:找一手文献(别集卷次/善本/碑帖),逐字核校原文,记 `ref`
3. **判级**:A/B/✗,写 `annotation` 说明确指依据
4. **白话**:写 `vernacular`
5. **定位**:geocheck.py 查 OSM 节点 → 得 WGS-84 坐标 + 溯源;查不到标 estimate
6. **归档**:写入对应区的 research/*.md,成熟后转 data/*.json
